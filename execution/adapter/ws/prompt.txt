Your task is a safe architectural refactor to unify execution semantics between FlightSQL and WebSocket servers WITHOUT introducing regressions.

Refactor the system so that:

All query execution logic is moved into a shared Engine abstraction

Both transports consume the same execution primitive:

BuildStream(ctx, sql, params) (*arrow.Schema, <-chan StreamChunk, error)


Concurrency control is shared via:

AcquireQuerySlot(ctx) / ReleaseQuerySlot()

Streaming semantics remain identical across both transports

change SQL semantics
change result ordering
change batching behavior
introduce additional goroutines in request paths
introduce buffering layers that alter backpressure
change cancellation semantics
duplicate execution logic between WS and FlightSQL
modify Arrow schema evolution behavior
change IPC encoding format unless explicitly required by shared abstraction
🧱 REQUIRED ARCHITECTURAL TARGET

After refactor, the system must follow:

                 ┌──────────────┐

                        │
                BuildStream(ctx,...)
